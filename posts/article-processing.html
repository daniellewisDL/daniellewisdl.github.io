Processing

January 2022

If you have a creative streak, and have wanted to explore how artists use code to generate interesting and amazing pieces of art, or if you are a coder, and want to use your skills to create something beautiful, then Processing is a great place to start.

Processing?

Yes, Processing has a name that can be confusing. "I made this image using Processing" is an odd phrase to someone who has not encountered the software before. But if you can get over the name, you will discover one of the greatest and easiest to use creative coding engines, that has been used to create all sorts of static and animated artwork, including pieces that have sold for tens of millions of dollars.

Processing is a group of related software projects including:

  • Processing - a desktop-based downloadable application using the Java programming language
  • p5.js - a JavaScript library that you can use to create Processing-like outputs directly in the browser
  • Processing.py - a python library implementing a version of Processing that can be used in python

The mission of the Processing Foundation, the organisation that oversees the projects to develop and maintain the software tools, is to "promote software literacy within the visual arts, and visual literacy within technology." If you ask me, that is a noble aim.

An example

The following image, or 'sketch' in the parlance of Processing, was generated by your browser when you loaded this page:


There are lots of ways of drawing this in Processing, but my p5.js version for the above is as follows:

                            const s1 = ( sketch1 ) => {

  let cs = 20;
  let x = 0-cs;
  let y;
  let offset=0;

  sketch1.setup = () => {
      sketch1.createCanvas(document.getElementById('first-sketch').offsetWidth, 100);
      y = sketch1.height/2;
      sketch1.background(255,255,255);
    };
    
  sketch1.draw = () => {
    sketch1.noLoop();
    while (offset <= 1) {

      sketch1.fill(255,255,255,10);
      sketch1.noStroke();

      if (x>(sketch1.width+cs)) { x = 0-cs; offset+=1;} else {x+=2;}
      x_prime = sketch1.map(x,0-cs,sketch1.width+cs,0,sketch1.TWO_PI);
      y = sketch1.map(sketch1.sin(x_prime+offset),-1,1,0+cs,sketch1.height-cs);

      sketch1.noFill();
      sketch1.stroke(0,0,0,20);
      sketch1.circle(x,y,cs);
    };    
  };
};

let sketch_1 = new p5(s1, 'first-sketch');

This might look like a lot of code for a relatively simple effect and so if you find it confusing, congratulations you are a human being. Do not be put off! This example is only meant to show that you can generate some interesting looking sketches using Processing (p5.js here) in the browser, in around 20 lines of code. This is not meant to be a full-blown tutorial either: the tutorials on the processing website are the best place to start for this. Frankly, there are probably many simpler and better ways to achieve the above effect, but by way of a brief explanation: my version counts up from 0 to the number of pixels wide, then for every count it draws a circle centred on the count for the x value, and for the y position it follows the sine function, to give the curvey shape. Once the count gets to the width of the canvas, it resets back to the left side, and offsetting the sine by a fixed increment, to make the curves overlap but not on top of each other. This version just does this twice, to display two curves. The key test while (offset <= 1) can be modified to draw additional curves. Generative Art

Processing can be used to produce all kinds of interesting and wonderful generative art. Generative art is experiencing a boom of interest at the moment with the rise of NFTs, but as is usually the case with the subjects of a hype-cycle, generative art is a field that has a long, rich and interesting history.
